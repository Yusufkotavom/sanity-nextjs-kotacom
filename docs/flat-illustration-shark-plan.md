## 1) Objective

Membangun sistem ilustrasi flat style yang konsisten untuk seluruh website KOTACOM, dengan:
- maskot utama karakter hiu (brand mascot)
- palet utama: biru, merah, kuning, hitam-putih
- visual modern, bersih, scalable untuk web
- setiap ilustrasi menggambarkan aktivitas bisnis secara jelas (bukan dekorasi abstrak saja)
- siap dipakai untuk hero, section layanan, contact, 404, CTA, dan state UI lain

## 2) Art Direction (Mandatory)

### Visual Style
- Flat vector illustration
- Geometric clean shapes, no photorealism
- Medium detail (cukup kaya elemen untuk storytelling bisnis)
- Minimal noise, no texture berat
- Soft shadow/ambient shadow tipis saja (opsional), tetap flat-first
- Rounded corners dan friendly proportions
- Reference feel: editorial flat illustration with human/mascot figure, UI panels, organic leaves/shapes, chat bubbles, and business props

Reference interpretation:
- Komposisi ringan dan lapang
- Satu scene = satu aktivitas jelas
- Character-driven, bukan dashboard-only
- Objek pendukung kecil memberi konteks tanpa membuat scene padat

### Tone & Brand Personality
- Profesional tapi approachable
- Enerjik dan solutif
- Menunjukkan “tim ahli yang membantu bisnis”
- Tidak childish/cartoon berlebihan

### Color System
- Primary Blue: `#0EA5E9`, `#0369A1`
- Accent Red: `#EF4444`, `#B91C1C`
- Accent Yellow: `#FACC15`, `#CA8A04`
- Support Neutral: `#0F172A`, `#334155`, `#94A3B8`, `#F8FAFC`, `#FFFFFF`

Rules:
- Background dominan netral (`#F8FAFC` / putih)
- Warna kuat dipakai untuk objek fokus (CTA object, icon, garis alur kerja, highlight tools)
- Kontras teks/objek tetap aksesibel

## 3) Mascot System - “KOTA SHARK”

### Core Character
- Karakter hiu ramah, cerdas, ekspresi percaya diri
- Bentuk sederhana: kepala hiu rounded, mata tegas tapi ramah, senyum ringan
- Tidak agresif (hindari gigi tajam dominan)
- Memakai elemen brand: headset/ID card/hoodie/jaket kerja sesuai scene

### Pose Library (reuse)
- Thinking/analysis pose (untuk konsultasi)
- Working at desk pose (untuk development/design)
- Presenting pose (untuk proposal/portfolio)
- Helping customer pose (untuk support/contact)
- Searching pose (untuk 404/empty state)
- Fast delivery pose (untuk SLA/proses)

### Consistency Rules
- Proporsi kepala:badan tetap konsisten antar ilustrasi
- Outline/line weight konsisten
- Style mata/mulut konsisten
- Jangan ganti species/karakter utama

## 5) Scene Detail Requirements (Per Illustration)

Setiap ilustrasi minimal harus punya:
- 1 maskot hiu utama
- 1 konteks bisnis jelas (alat, dashboard, mesin, dokumen, customer, packaging, dsb)
- 1 alur tindakan (mis. konsultasi -> desain -> produksi -> delivery)
- 2-4 objek pendukung yang relevan
- ruang kosong aman untuk overlay headline/CTA (safe area)

Safe layout:
- desktop: area kosong teks di kiri 35-45% atau kanan 35-45%
- mobile crop: elemen penting tetap terlihat di tengah

## 5.1 Remodeling Directive (Mandatory)

Remodeling berarti tidak hanya mengganti gambar, tetapi menyegarkan experience section agar:
- lebih engaging
- tidak padat/over-information
- CTA terlihat jelas pada interval scroll

Remodeling checklist khusus `jasa-cetak-buku`:
1. Hero:
- gunakan hero visual yang relevan ke cetak buku (workflow nyata)
- tampilkan branding KOTACOM secara halus (mengacu logo resmi)
- tetap sisakan safe area untuk headline + CTA utama

2. Section rhythm:
- sisipkan visual pada blok layanan, proof, dan CTA
- jaga whitespace antar section (jangan terlalu rapat)
- maksimal 1 fokus visual utama per section agar tidak noisy

3. CTA cadence:
- CTA utama di hero
- CTA sekunder di tengah (quick actions)
- CTA final di akhir halaman
- setiap CTA harus memiliki konteks manfaat singkat

4. Trust and detail:
- tampilkan elemen proses cetak (proofing, jilid, quality control, pengiriman)
- tampilkan indikator profesional (garansi, timeline, support)

5. Consistency:
- mascot hiu konsisten
- warna konsisten dengan sistem brand
- gunakan logo resmi hanya sebagai brand cue, bukan watermark dominan

## 8) Production Workflow

1. Generate rough batch (low fidelity) untuk 8 use-case utama.  
2. Pilih visual direction terbaik per use-case.  
3. Regenerate high-detail final assets (consistent mascot).  
4. Export final in:
- SVG master
- PNG fallback 2x (`1600x900` / `1200x675`)
5. Integrasikan ke komponen per section.  
6. Uji responsive crop desktop/mobile.
